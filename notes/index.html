    • Notes2ppt

    When I started working on Notes2ppt, It was in a very juvenile state.Actually this project deals with Optical character recognition that is extracting tect from image .It's motive is to build ppts from hand written notes,using Tesseract.Pre image processing is done through Opencv.PPt's are generated through python pptx & wiki api.Pyenchant & NLTK is used for spell-checking.

    Gunjan Sengupta has described the working of Notes2ppt in a very simple manner & fluent manner.Anyone can follow the instruction easily.But setting up the condition to work on was quite difficult.

    Basic working of Notes2ppt

    First Canny edge detector is applied to find edges

    Then by using approx poly DP contours are detected,then From those contours, contour with biggest area is selected

    Image is processed for text extraction

    Now tesseract engine extracts text

    OS package is imported to run spellcheck.py to remove spelling mistakes & delete warped.png(os.remove)(black image with white text)

    This is interpreter demonstration of Spellchecker(Handwritten package)

    Problems faced

    Most of the problem was getting solved by googling but there was one problem which I was not able to solve myself as I was not able to understand the error.When I was tokenizing the raw text extracted by tesseract,terminal was continuously giving ascii error.As I was not fluent in tokenizers I was trying to find fault in that function.But after my long wasted effort I was told to use this to remove ascii error "tokens+=nltk.word_tokenize(re.sub(r'[^\x00-\x7F]+',' ', line))"
